Onion Architecture Layers

This application is structured using the Onion Architecture with the following layers:

  1. Common - Shared utilities and cross-cutting concerns
  2. Domain - Pure business logic (core)
  3. Enactment - Application services and use cases
  4. Infrastructure - External concerns (databases, APIs)
  5. Integration - Framework adapters
  6. Presentation - UI layer (this component!)
